# Phase 3: 精修审查

## 进入条件
- 全书完成 或 整卷完成
- 或作者说「审稿」「精修」「润色」「去味道」「检查一下」

## 流程

### Step 1: 全局审查

**执行者：** consistency-checker Agent（加载 `references/agents/consistency.md`，深度模式）

```
全篇四维扫描：
  ├── 事实一致性：角色/伏笔/时间线/空间，全篇交叉比对
  ├── 逻辑一致性：因果链/动机链/规则自洽/信息传递
  ├── 结构一致性：节奏/信息密度/钩子密度/篇章比例
  └── 缺口扫描：大纲缺失/设定空白/伏笔断线

输出：审查/全局审查报告.md、审查/一致性检查.md
```

### Step 2: 分级处理协商

```
「全局审查完成。
  🔴 必须修：N 项（逻辑断裂/严重矛盾/角色 OOC）
  🟡 建议修：M 项（不够合理/节奏问题/信息过载）
  🟢 供参考：P 项（优化建议）

  要逐项讨论还是先集中处理 🔴？」
```

处理模式由作者选择：
- **逐项模式**：每个 🔴 问题依次展示 → 提修正方案 → 作者决定 → 执行
- **批量模式**：AI 批量修复所有 🔴🟡 → 展示改动 → 作者确认
- **混合模式**：🔴 逐项处理，🟡 批量修复

### Step 3: 文风统一

**执行者：** polisher Agent（加载 `references/agents/polisher.md`）

```
五维精修：
  1. 去 AI 味（三遍法）
  2. 文风统一（视角/时态/距离/文白度）
  3. 对话优化（辨识度/功能性/节奏/潜台词）
  4. 节奏打磨（段落/句式/比例）
  5. 禁用词扫描

每维度完成后展示修改 → 与作者协商 → 确认后继续下一维度

输出：审查/文风检查.md
```

### Step 4: 审查汇总

```
所有审查文件已产出：
  - consistency-checker：审查/全局审查报告.md、审查/一致性检查.md
  - polisher：审查/文风检查.md

所有文件放在项目的 审查/ 目录下
```

### Step 5: 精修完成

```
「精修阶段完成。审查报告在 审查/ 目录下。
  共修复：
  - 🔴 必须修 N 项
  - 🟡 建议修 M 项
  - 文风优化 P 处
  
  还有什么想再调整的吗？」
```

---

## 分段精修（按卷）

如果全书较长，可按卷分段精修：

```
第 1 卷完成 → 全局审查（第 1 卷范围）→ 精修 → 继续写第 2 卷
```

这样避免后期审查时发现问题积累太多。

---

## 特别注意

- 去 AI 味不是去文采——保留好的描写，替换套话
- 精修阶段不需要重写大纲，除非发现严重结构问题
- 不确定的修改一定先问作者
- 修改要展示原句→改句，让作者看到动了哪里
- 如发现需要大规模重写的问题（>30% 内容），先与作者协商是否值得
